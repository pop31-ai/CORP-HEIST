4. INTER-INSTANCE COMMUNICATION
===============================

  Old model (30 processes):
    Process A вҶҗвҶ’ IPC/Redis вҶҗвҶ’ Process B
    Overhead: serialization + context switch + memory copy

  New model (1 process, SharedState):
    Instance A вҶҗвҶ’ SharedState вҶҗвҶ’ Instance B
    Overhead: dict reference (0 bytes)
    Latency: 0ms (same memory)
    Traffic: 0 bytes

7. SCALING (if needed)
=======================

  To scale beyond 500 chars:
  - Increase NUM_CHARS (up to 10K in same process)
  - Each char ~2KB, 10K chars = 20MB
  - Still under 100MB total

  To scale to multiple VMs:
  - Add Redis for char/market state sync
  - Each VM runs consolidated server
  - Shared Redis = shared market
  - Cross-VM P2P = 0 (still zero P2P)
